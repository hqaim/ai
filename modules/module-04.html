Module 04 — CLARITY-7™: The Intent Design Framework

Master the CLARITY-7™ framework to design precise, reliable AI prompts.

Key Insight

Clarity is not accidental; it is designed. The CLARITY-7™ framework forces you to define every aspect of your intent, leaving no room for ambiguity.

Why This Matters

Without a structured approach, prompts are often vague or incomplete, leading to unpredictable results. By defining all seven fields, you create a clear contract with the AI model.

This module delves into each field of the CLARITY-7™ framework, explaining its purpose and how to use it effectively. Mastering this framework is the key to becoming an AI Artisan.

Framework Foundations

The CLARITY-7™ framework consists of seven essential fields that must be defined for every prompt:

  • 1. Purpose: The primary goal or objective.
  • 2. Audience: The intended recipient or user.
  • 3. Constraints: Limitations, boundaries, or requirements.
  • 4. Format: The structure or layout of the output.
  • 5. Tone: The style, voice, or mood.
  • 6. Inputs: Any data, text, or context provided.
  • 7. Expected Output: A clear description of the final result.

Core Ideas Explained

Why Intent Must Be Explicit

AI does not access hidden context. It only sees what the user writes. If intent is not fully defined, the model fills missing gaps using default assumptions. These assumptions often differ from the user’s goals. CLARITY-7™ forces the intent to be explicit and complete.

The Seven Fields Create Stability

Each field provides a different type of clarity. Purpose explains why. Audience clarifies who it is for. Format and tone shape the output. Constraints define boundaries. Inputs organize information. Expected output describes the final result. When these fields are present, the model produces stable and predictable responses.

Why Fields Must Be Separate

Combining fields in one sentence produces ambiguity. For example, mixing tone with purpose hides important details. Separating the fields forces precision and eliminates confusion.

Structure Before Generation

Many users write prompts while thinking through the task. This leads to unclear instructions. CLARITY-7™ separates thinking from generating. First, define the structure. Then generate the output. This improves clarity and reduces editing time.

Reuse and Expansion

A well-designed CLARITY-7™ prompt can be reused across tasks. As needs evolve, the fields can be updated or expanded without rewriting the prompt from scratch. This makes it a long-term tool, not a one-time technique.

How the Framework Applies

This module explains how to design prompts that are complete, stable, and unambiguous.

CLARITY-7™ is applied by filling out each field separately. This forces the user to think clearly about purpose, boundaries, and the expected result. SPARK-5™ is used to refine individual fields before generating.

After generation, MIRROR-6™ verifies whether the output matches the seven fields. If it does not, the user updates the fields and regenerates.

This approach creates a clean cycle of design, refinement, and verification that produces reliable outcomes.

Step-by-Step Method

This method shows how to apply CLARITY-7™ before generating content.

  1. Define your purpose.
    Write a short statement describing what you want to achieve.
  2. List your audience.
    Identify who will read or use the output.
  3. Describe constraints.
    Include rules, boundaries, or exclusions.
  4. Decide on the format.
    Specify the structure you want, such as bullet points or paragraphs.
  5. Set the tone.
    Choose a tone that fits your goal, such as formal or neutral.
  6. Prepare inputs.
    Organize documents, links, or notes needed for the task.
  7. Define your expected output.
    Describe what the model should produce and why.
  8. Run SPARK-5™ if anything feels unclear.
    Improve fields before generating.
  9. Generate and verify.
    Use MIRROR-6™ to confirm accuracy and alignment.

Typical Mistakes & Fixes

Combining fields into one sentence

This hides important details.

Fix: Keep each field separate and specific.

Ignoring constraints

Lack of constraints leads to generic results.

Fix: Add limits, exclusions, and boundaries.

Forgetting the audience

Outputs must fit the reader’s needs.

Fix: Describe audience knowledge, experience, or role.

Undefined expected output

Without it, the model guesses what to produce.

Fix: Specify exact deliverables and purpose.

Starting from zero each time

This wastes time and reduces consistency.

Fix: Save field sets using TESSERA™.

How to Check Your Results

MIRROR-6™ provides the main verification method. Check for:

  • Meaning: Does the result follow the purpose?
  • Accuracy: Are statements correct?
  • Relevance: Does it match the seven fields?
  • Reliability: Could this be reproduced consistently?
  • Outcome Alignment: Does it support your intended result?
  • Repeatability: Can someone else follow the steps?

If multiple gates fail, refine your CLARITY-7™ fields first.

If the output passes but feels low value, use VALOR-Δ to measure improvement and identify weaknesses.

Key Takeaways

  • CLARITY-7™ removes ambiguity and defines complete intent.
  • Separate fields create precise, stable prompts.
  • SPARK-5™ refines individual fields before generating.
  • MIRROR-6™ checks alignment and accuracy after generation.
  • Clear intent reduces guesswork and improves outcomes.
  • Structured design is essential for dependable AI work.
