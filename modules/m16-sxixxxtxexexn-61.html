Module 16 — Advanced Prompt Patterns

Master complex reasoning structures for consistent, high-level outputs.

Key Insight

Advanced prompt patterns guide the model through defined reasoning paths. They help it separate stages, follow structure, and produce reliable results for tasks that need more than a single instruction.

Why This Matters

Basic prompting is unstable for tasks involving comparison, analysis, evaluation, or transformation. These tasks require clear reasoning flows. Without structure, the model blends steps or skips important logic. This results in errors or inconsistent quality.

Advanced patterns solve this by creating predictable pathways. Each pattern defines stages, boundaries, and checks. Professionals benefit because patterns reduce variability across projects. Beginners benefit because patterns reduce guesswork and offer stable templates for complex work.

Framework Foundations

  • CLARITY-7™ defines purpose, audience, constraints, tone, input format, and expected output. It stabilizes each pattern and ensures every stage has a clear objective.
  • MIRROR-6™ checks meaning, accuracy, relevance, reliability, outcome alignment, and repeatability. It verifies whether each stage stays aligned with the intended reasoning path.
  • VALOR-Δ measures whether the pattern provides measurable improvements across attempts. It evaluates value, alignment, leverage, outcome strength, repeatability, and change (Δ).
  • TESSERA™ stores patterns as continuity tiles so they can be reused, refined, and adapted without rebuilding.

These frameworks turn patterns into durable reasoning tools.

Core Ideas Explained

Why Patterns Strengthen Reasoning

Reasoning often breaks when tasks rely on a single instruction. The model attempts to compress multiple steps into one output, which leads to drift or omissions. Patterns solve this by providing separate stages with defined goals.

Each stage limits ambiguity. CLARITY-7™ defines what to do, how to do it, and what the output should contain. This improves precision and reduces misalignment across stages.

Pattern Types for Structured Thinking

Different tasks require different reasoning structures. Analytical tasks benefit from staged breakdown patterns. Creative tasks benefit from divergent–convergent structures. Decision tasks require criteria-based comparison patterns.

Each pattern uses clear inputs, step boundaries, and output formats. This standardizes the reasoning process and makes outcomes predictable across platforms.

Managing Complexity with Pattern Chains

Some tasks require more than one pattern. Pattern chaining creates multi-step flows where each pattern supports a different part of the process: idea generation, evaluation, synthesis, or transformation.

Chaining allows users to combine strengths of different patterns while preserving stability. MIRROR-6™ checks each pattern in the chain. VALOR-Δ measures whether chaining improves results.

Step Separation for High-Stability Tasks

Complex reasoning becomes unreliable when steps merge or overlap. Advanced patterns enforce step separation. The model completes one stage fully before starting the next. This protects the reasoning path and reduces mistakes.

Step separation also makes verification easier. When a stage fails MIRROR-6™, users can refine that stage alone rather than repeating the entire process.

Reuse and Refinement through TESSERA™

Patterns that produce strong results should become long-term assets. TESSERA™ stores refined patterns, examples, proof notes, and Δ findings. This preserves successful designs for future use.

This continuity reduces rebuilding and allows for consistent reasoning across tasks, teams, and projects.

How the Framework Applies

CLARITY-7™ defines the intention and boundaries of each pattern. It specifies purpose, audience, inputs, constraints, tone, and expected output. This ensures the model enters each stage with a stable goal.

MIRROR-6™ verifies accuracy across each stage. It checks meaning, accuracy, relevance, reliability, outcome alignment, and repeatability. This prevents weak stages from influencing later steps.

VALOR-Δ measures whether a pattern improves performance compared to a basic prompt. If Δ is strong, the pattern is effective. If Δ is weak, the pattern structure needs refinement.

TESSERA™ stores refined patterns so they can be reused consistently without rebuilding. This builds a library of stable, high-value reasoning tools.

Pattern Forge Prompt 1 — Multi-Stage Reasoning Pattern Builder

Multi-Stage Reasoning Pattern Builder 
ROLE: Reasoning Pattern Architect
CONTEXT: You need a pattern that guides complex reasoning across
         separate stages. CLARITY-7™ defines purpose, inputs,
         constraints, tone, and expected output.
TASK: Build a reasoning pattern with 4 clear stages:
      1) Understanding the Task
      2) Structured Breakdown
      3) Step-by-Step Reasoning
      4) Final Synthesis
CONSTRAINTS: Keep each stage independent; add one sentence
             explaining the goal of each stage.
OUTPUT: A complete multi-stage reasoning pattern ready for reuse.
Usage Guide

When to use: For tasks that require analysis, breakdown, and synthesis.

Problem it solves: Models blend steps or skip reasoning.

How to use: Provide your task under the prompt.

Good output: Four labeled stages with clear goals.

Why it works: CLARITY-7™ ensures each stage has defined intent.

Beginner benefit: Makes complex reasoning easy to follow.

Pattern Forge Prompt 2 — Divergent–Convergent Expert Pattern

Divergent–Convergent Expert Pattern 
ROLE: Exploration Pattern Designer
CONTEXT: Some tasks require broad exploration before narrowing down.
         MIRROR-6™ verifies the quality of the final choice.
TASK: Create a pattern with two phases:
      A) Divergent generation of options (3–5 options)
      B) Convergent selection using three MIRROR-6™ criteria
CONSTRAINTS: Options must be independent; criteria must be explicit;
             selection must be justified.
OUTPUT: A structured divergent–convergent reasoning pattern.
Usage Guide

When to use: When you need multiple options and a structured way to choose one.

Problem it solves: Options become similar or selection becomes subjective.

How to use: Add your topic and run both phases.

Good output: A range of options and a justified selection.

Why it works: MIRROR-6™ creates objective evaluation.

Beginner benefit: Encourages exploration with guided decision-making.

Pattern Forge Prompt 3 — Transformation Pattern

Transformation Pattern 
ROLE: Transformation Engineer
CONTEXT: Content transformations need accuracy and consistency.
         CLARITY-7™ defines inputs and outputs. MIRROR-6™ maintains alignment.
TASK: Build a transformation pattern with three steps:
      1) Interpret the source
      2) Adapt to new constraints
      3) Deliver the final formatted output
CONSTRAINTS: No loss of meaning; define input and output formats;
             no merging of steps.
OUTPUT: A structured transformation pattern.
Usage Guide

When to use: When rewriting, summarizing, adapting, or reformatting content.

Problem it solves: Transformations lose detail or drift in tone.

How to use: Insert source content and target format.

Good output: Accurate, aligned, and clearly formatted output.

Why it works: MIRROR-6™ ensures fidelity to original meaning.

Beginner benefit: Produces clean transformations without expertise.

Pattern Forge Prompt 4 — Δ-Driven Decision Pattern

Δ-Driven Decision Pattern 
ROLE: Decision Framework Analyst
CONTEXT: Complex decisions need transparent criteria. VALOR-Δ
         measures value, alignment, leverage, outcome strength,
         repeatability, and improvement.
TASK: Create a decision pattern with:
      - Three options
      - Four evaluation criteria
      - A Δ-based improvement assessment
CONSTRAINTS: Scoring must be simple; avoid vague judgments; include
             one actionable refinement.
OUTPUT: A decision pattern with transparent scoring and a final choice.
Usage Guide

When to use: When deciding between strategies, actions, or plans.

Problem it solves: Decisions become subjective or inconsistent.

How to use: Provide three options and run the evaluation.

Good output: A justified decision with a clear Δ-based rationale.

Why it works: VALOR-Δ highlights measurable improvements.

Beginner benefit: Helps users make decisions with confidence.

Step-by-Step Method

This process ensures consistent, high-quality reasoning.

  1. Define the task using CLARITY-7™.
  2. Select the pattern type that fits the task.
  3. Apply each stage of the pattern without merging steps.
  4. Verify each output using MIRROR-6™.
  5. Compare results across runs using VALOR-Δ.
  6. Refine unclear stages or constraints.
  7. Save the improved pattern as a TESSERA™ tile.
  8. Reuse the pattern for similar tasks.

Typical Mistakes & Fixes

Using patterns without clear intent

Fix: Define the task using CLARITY-7™ first.

Merging steps in reasoning patterns

Fix: Keep stages separate and verify each one independently.

Lack of evaluation criteria

Fix: Use MIRROR-6™ or simple scoring rules.

Overcomplicating pattern structures

Fix: Limit each stage to a single function.

Not measuring improvement

Fix: Use VALOR-Δ to track gains across attempts.

How to Check Your Results

Use MIRROR-6™ to verify each stage for meaning, accuracy, relevance, reliability, outcome alignment, and repeatability. If a stage fails, refine its constraints or rewrite the step.

Use VALOR-Δ to measure improvement across repeated uses. Strong Δ indicates the pattern works. Weak Δ signals unclear stages or poor constraints. Review Proof Artifacts™ to confirm alignment.

When both checks show stability, store the pattern using TESSERA™.

Short Practice Tasks

  1. Apply the Multi-Stage Reasoning Pattern to a current task and compare its output to a basic prompt.
  2. Use the Divergent–Convergent Pattern to generate options for a decision and verify the selection using MIRROR-6™.
  3. Run the Transformation Pattern on the same input twice and track Δ improvements using VALOR-Δ.

Key Takeaways

  • Advanced patterns improve control and reduce reasoning drift.
  • CLARITY-7™ stabilizes each stage of the pattern.
  • MIRROR-6™ verifies accuracy and alignment at every step.
  • VALOR-Δ measures the real improvement gained from using patterns.
  • TESSERA™ stores refined patterns for long-term reuse.
  • Step separation increases consistency and clarity for complex tasks.
