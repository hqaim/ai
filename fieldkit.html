Toolkit — Resources & Templates

Structured tools for the AI Artisan.

How to Use This Toolkit

Workflow

  • Use templates to start tasks with clarity.
  • Use checklists to review drafts systematically.
  • Use scorecards to verify accuracy and measure value.
  • Use logs to store your final prompts and outputs for reuse.

Integration

Combine tools as needed depending on your workflow. These resources are designed to work independently or as a connected system (TESSERA™).

Checklists (SPARK-5™)

Checklists reduce blind spots. Use these principles to refine clarity, alignment, structure, and consistency.

4.1 Pre-Prompt

  • □ Purpose defined
  • □ Audience specified
  • □ Constraints added
  • □ Format selected
  • □ Success criteria set

4.2 Post-Draft

  • □ Irrelevant content removed
  • □ Structure consistent
  • □ Tone correct
  • □ Examples accurate
  • □ Steps complete

4.3 Context Completeness

  • □ Background included
  • □ Domain rules stated
  • □ Edge cases noted
  • □ Scope defined
  • □ Expected boundaries set

Scorecards

Evaluate quality and value using MIRROR-6™ (Accuracy) and VALOR-Δ (Value).

5.1 Accuracy (MIRROR-6™)

Rate 1–5 (Pass/Fail)

  • Meaning: Matches intent?
  • Integrity: Correct facts?
  • Relevance: On topic?
  • Reliability: Reproducible?
  • Outcome alignment: Fits goal?
  • Repeatability: Process clear?

5.2 Value (VALOR-Δ)

Rate 1–5

  • Value: Utility
  • Alignment: Fit
  • Leverage: Efficiency
  • Outcome Quality: Impact
  • Repeatability: Stability
  • Δ Improvement: Gain over manual

5.3 Workflow Quality

Checklist

  • Clarity of steps
  • Logic flow
  • Checkpoints present
  • Efficiency
  • Completeness
  • Adaptability

Field Tools

Small, fast tools for everyday use.

7.1 Pattern Selector

Choose the right reasoning pattern:

  • Divergent Generation: For brainstorming.
  • Convergent Selection: For narrowing options.
  • Multi-Stage Reasoning: For complex logic.
  • Constraint-Driven Shaping: For specific formats.
  • Transformation Steps: For rewriting/formatting.

7.2 Workflow Builder

Step-by-step creator for common task structures. Supports planning, sequencing, and defining checkpoints.

7.3 Verification Protocol

A simplified MIRROR-6™ form for quick checks. Extremely useful during fast iteration cycles.

7.4 Δ Tracker

A compact VALOR-Δ tracker for recording improvements across versions. Focuses on the "Delta" (Change).

Practical Examples

8.1 Using an Intent Template

  1. Fill the CLARITY-7 fields for a writing task.
  2. Generate a first draft.
  3. Refine using SPARK-5™.
  4. Verify with MIRROR-6™.

8.2 Using a Scorecard

  1. Apply MIRROR-6™ to a product description.
  2. Document scores (1-5).
  3. Improve the weak areas.
  4. Re-score to measure Δ.

8.3 Using a Log

  1. Complete a task.
  2. Capture the final prompt and output.
  3. Add notes on what worked.
  4. Save for future reuse.

Best Practices

  • Keep templates simple and consistent.
  • Avoid unnecessary fields.
  • Use checklists before and after generation.
  • Score outputs regularly to build accuracy habits.
  • Store every major prompt and workflow.
  • Reuse successful tools instead of rebuilding them.
